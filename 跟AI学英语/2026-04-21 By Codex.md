## 2026.04.21

### Graduate / Grad Student

- `graduate student` / `grad student` = 研究生，通常指正在读硕士或博士的人。
- `graduate` = 毕业生，指已经完成某个学位的人，不等于“研究生”。
- `graduated` = 已经毕业了，是 `graduate` 的过去式。
- `graduate school` = 研究生院 / 研究生阶段。
- `postgraduate` = 研究生 / 研究生阶段，英式英语更常见；美式英语更常说 `graduate student`。

Examples:

- `I’m a graduate student.` = 我是研究生。
- `I’m a college graduate.` = 我是大学毕业生 / 本科毕业生。
- `I graduated from college in 2024.` = 我 2024 年大学毕业。
- `She plans to go to graduate school next year.` = 她计划明年读研。

Common mistake:

- 不要用 `I am a graduate.` 表达“我是研究生”。这句话更像“我是毕业生”。

### Coursework And Assignments

- `coursework` = 课程任务、课程作业、课程相关工作。
- `assignments` = 作业、任务。
- `coursework and assignments` 比 `courses and assignments of courses` 更自然。
- `obstacle` = 阻碍、障碍。
- `get in the way of ...` = 妨碍……、挡着……。

Examples:

- `Coursework and assignments are the biggest obstacles to graduate students' research and work.`
- `Coursework is the main thing getting in the way of grad students' research and work.`

Nuance:

- `graduate students` 比 `grad students` 更正式。
- `grad students` 更口语、更自然，适合聊天或 X。

### Lock In

- `lock in` = 进入专注状态、认真投入、开始专心干活。
- 这是比较口语的表达，常见于学习、工作、健身语境。

Examples:

- `Play a playlist to help me lock in.` = 放个歌单，帮我进入专注状态。
- `I need to lock in for this exam.` = 我得为这场考试认真专注起来。
- `Let’s lock in and finish this project.` = 我们专心把这个项目做完吧。
- `I’m locked in right now.` = 我现在很专注 / 已经进入状态了。

Similar phrase:

- `get in the zone` = 进入状态。

Example:

- `Play something that helps me get in the zone.`

### Commit Message Expressions

- `error handling` = 错误处理，不常说 `error handle`。
- `terminal output` / `console output` / `CLI output` = 终端输出 / 命令行输出。
- `in a commit message` = 在一条 commit message 里；注意要有 `a`。

Examples:

- `fix: improve error handling and terminal output`
- `fix: handle command errors more gracefully`
- `feat: add clearer terminal output for failed commands`
- `chore: clean up CLI logging`

For “扩充提取得到的 CSV 数据的结果列”:

- `feat: add result columns to CSV extraction output`
- `feat: expand result columns in extracted CSV data`
- `fix: include more result columns in extracted CSV data`
- `feat: extend CSV extraction result columns`

Nuance:

- `CSV extraction output` 往往比 `extracted CSV data` 更适合 commit message。
- 技术写作里通常写 `CSV`，不要写 `csv`。

### And vs &

- `and` 用在普通句子里。
- `&` 多用于短标题、标签、UI 文案、表格、公司名、固定搭配，有时也可用于 commit message。

Good uses:

- `Research & Development`
- `Settings & Privacy`
- `Terms & Conditions`
- `Error Handling & Terminal Output`

Normal sentence:

- Better: `I study AI and build apps.`
- Less natural: `I study AI & build apps.`

Simple rule:

- Full sentence: use `and`.
- Short label/title/UI/commit heading: `&` is okay.

### View / Opinion / Take

`观点` can be translated in different ways:

- `opinion` = 个人看法。
- `view` = 看法，比较通用。
- `point of view` = 观点 / 视角。
- `perspective` = 视角、角度。
- `argument` = 论点，常用于论文或辩论。
- `claim` = 主张，常用于学术写作。
- `position` = 立场。
- `take` = 口语里的“看法 / 观点”。
- `thoughts` = 想法 / 看法，很自然、友好。

Oral / spoken expressions:

- `What’s your take on this?` = 你怎么看这个？
- `My take is that coursework takes up too much time.` = 我的看法是，课程作业太占时间了。
- `That’s an interesting take.` = 这个观点挺有意思。
- `Hot take: exams are not the best way to measure learning.` = 大胆说一句 / 可能有争议的观点：考试不是衡量学习的最好方式。
- `What are your thoughts on this?` = 你对这个有什么看法？
- `What do you think?` = 你怎么看？最常见、最自然。

### Align On Requirements

`对齐一下需求` can be translated as:

- `Let’s align on the requirements.` = 我们对齐一下需求。
- `Let’s align on the requirements first.` = 我们先对齐一下需求。
- `Let’s sync on the requirements.` = 我们同步一下需求。
- `Let’s clarify the requirements first.` = 我们先把需求说清楚。
- `Let’s make sure we’re on the same page about the requirements.` = 我们确认一下大家对需求理解一致。

Nuance:

- `align on the requirements` = 工作场景里很自然。
- `sync on the requirements` = 团队内部更口语。
- `clarify the requirements` = 强调把不清楚的地方讲清楚。
- `on the same page` = 理解一致，很地道。

### X / Casual Writing

For casual posts on X, shorter and punchier is usually better.

Draft:

```text
I think today’s AI evals miss something important.

Benchmarks test correctness and whether code passes tests, but rarely whether the model understood the user’s real intent, context, and constraints.

AI can write beautiful code and still miss what I actually need.

Maybe that’s partly my prompting, but after billions of tokens, I still feel this gap.
```

Useful phrases:

- `miss something important` = 漏掉了重要的东西。
- `benchmarks` = 评测 / 基准测试。
- `correctness` = 正确性。
- `passes tests` = 通过测试。
- `user’s real intent` = 用户真实意图。
- `context and constraints` = 背景和约束。
- `miss what I actually need` = 没抓住我真正需要什么。
- `burn through billions of tokens` = 消耗了数十亿 token，很口语。
- `feel this gap` = 感受到这种差距 / 落差。
- `dig into it` = 深入研究一下。

### And Still vs But Still

Sentence:

- `AI can write beautiful code and still miss what I actually need.`

Meaning:

- AI 可以写出很漂亮的代码，但仍然可能没抓住我真正需要的东西。

Nuance:

- `and still` = 即使这样，仍然……；听起来更自然、更有张力。
- `but still` = 也正确，更直接、更明确地表达转折。

Both are correct:

- `AI can write beautiful code and still miss what I actually need.`
- `AI can write beautiful code but still miss what I actually need.`

### Useful Question Patterns

- `Was my question grammatically correct?` = 我的提问语法正确吗？
- `Did I ask that correctly?` = 我刚才那样问对吗？更口语。
- `How can I say this in English?` = 这个用英语怎么说？
- `How can I express this in a commit message?` = 这个在 commit message 里怎么表达？
- `How should I phrase this in a commit message?` = 这个在 commit message 里应该怎么措辞？更自然。
- `How do you translate 观点?` = 观点怎么翻译？
- `How can I translate 观点 into English?` = 我怎么把“观点”翻译成英文？
- `Would "but" be better than "and" in this sentence?` = 这句话里用 but 会不会比 and 更好？

Small grammar notes:

- `Does graduate include grad student?` 比 `Dose graduate include grad student?` 正确。
- `Does "graduate" include "grad student"?` 更清楚，因为是在讨论词本身。
- `How can I express ... in a commit message?` 注意 `a commit message`。
- `How should I phrase ...?` 在问措辞时非常自然。
